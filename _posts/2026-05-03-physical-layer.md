## 问题 1：0 和 1 长什么样？

电脑里的 0 和 1 是抽象概念——晶体管导通/截止状态。要把它们发到 100 米外的另一台电脑，必须转换成真实物理世界的东西。

三种物理信号可选：

```
电压（网线）：    高电压 = 1，低电压 = 0
光脉冲（光纤）：  有光 = 1，无光 = 0
电磁波（Wi-Fi）： 用不同频率或相位表示 0 和 1
```

选了介质之后，下一个问题立刻出现：**信号到了对端还认得出来吗？**

---

## 问题 2：信号在路上被毁掉了

你发的是干净的方波。对方收到的是揉烂的、沾满噪声的东西。

**为什么会变形？——衰减。**

电信号在铜线里传导 → 铜有电阻 → 电能转化为热能 → 信号越来越弱。5V 起点，100 米后剩下 2V。再远，信号弱到跟背景噪声分不出。网线最长 100 米，不是"标准规定的"，是物理不允许。

**为什么有噪声？——热噪声。**

高于绝对零度的任何物质，原子都在振动。导体里的电子随之无规则运动 → 导线上出现随机的微小电压波动。热噪声无法消除（除非把设备泡液氮里），它是宇宙的基本物理规律。

**衰减 + 噪声 = 发送和接收的巨大差异：**

```
发送方：干净的方波
      ┌──┐    ┌──┐
      │  │    │  │      ← 0 和 1 边界清晰
──────┘  └────┘  └──

接收方：揉烂的、叠了噪声的波形
        ╱╲      ╱╲
       ╱  ╲    ╱  ╲    ← 边界模糊，叠加了随机毛刺
──────╱    ╲──╱    ╲──
    上一个比特  这一个比特
    的尾巴      到底是 0 还是 1？
    还在拖
```

这个问题直接引出下一个问题：**在一条有噪声的线路上，每秒最多能无误地发多少比特？**

## 问题 3：香农定理——不可突破的物理上限

香农 1948 年的答案：

```
C = B × log₂(1 + S/N)

C   = 信道容量（bps），这条线路每秒最多能无误传输的比特数
B   = 物理带宽（Hz），这条线路能用的频率范围有多宽
S/N = 信号功率 ÷ 噪声功率（一个倍数，不是 dB）
```

### 先理解 B——"你能用多大的频率范围"

频率范围越宽 → 单位时间内信号能变化的次数越多。

- 电话线：约 300Hz ~ 3400Hz → B ≈ 3100Hz（只要听懂说话内容，不需要 Hi-Fi 音质）
- Wi-Fi 2.4GHz 一个信道：20MHz
- 光纤 C 波段一个波长窗口：THz 级别

### 再理解 S/N——"信号和噪声的嗓门对比"

- S/N = 1000（30dB）→ 安静办公室里，正常说话 vs 空调嗡嗡声
- S/N = 1（0dB）→ 工地上，正常说话 vs 焊机轰鸣。谁也听不清你

### 然后理解 log₂(1+S/N)——"一次信号变化能塞几个比特"

噪声在信号上叠了一层模糊值。你想在一次信号变化里区分更多种状态，就需要相邻状态之间的间隔比噪声更大。

```
S/N = 1（信号和噪声一样强）
  → 只能可靠区分 2 种状态（0 和 1）
  → log₂(2) = 1，每次变化带 1 个比特

S/N = 7  → 区分 8 种状态 → 每次带 3 个比特
S/N = 15 → 区分 16 种状态 → 每次带 4 个比特
S/N = 1023 → 区分 1024 种状态 → 每次带 10 个比特
```

**规律：信号越强、噪声越弱，每次信号变化就能塞更多比特。** 但 S/N 翻倍，每次能带的比特数只增加不到 1 个（对数增长极慢）。这就是为什么 B 比 S/N 更值钱——B 翻倍，C 直接翻倍；S/N 翻倍，C 几乎没涨。

### 代入完整公式

```
普通电话线：
  B = 3100 Hz，S/N = 1000（30dB）
  C = 3100 × log₂(1+1000) ≈ 31,000 bps
  → 拨号上网不超过 56Kbps 的物理原因

光纤：
  B = 数十 THz（比电话线宽 10,000,000 倍以上）
  C = 极大
  → 光纤碾压铜线，不是因为"光信号强"，而是因为 B 大了几个数量级
```

### 香农定理给出的三个硬约束

1. **B 比 S/N 更值钱**。B 翻倍 ＝ C 翻倍（线性）；S/N 翻倍 ＝ C 涨不到 1 比特（对数）。
2. **速率有不可突破的上限**。任何编码方案只能逼近香农极限，不能超越。
3. **噪声永远存在**。热噪声 N = kTB（k 是宇宙常数，T 是温度，B 是带宽）。B 增大了 N 也跟着增大。

## 问题 4：时钟同步——接收方怎么知道每个比特从哪开始？

A 每秒发 100 万个比特。B 怎么知道这个节奏？

A 和 B 各有一个晶振（石英钟）。两个晶振的频率不可能完全一样。一个可能是 1,000,000.1 Hz，另一个是 1,000,000.0 Hz。10 秒后偏差累积到 1 个比特周期 → B 的采样点完全错位 → 读到的是垃圾。

时钟漂移不是"设备坏了"，是**两个物理晶振必然有的微小差异**。

### 方案一：加一根独立的时钟线（短距离可行，长距离不行）

两根线长度不可能完全一样 → 长距离下，时钟和数据信号的到达时间出现偏差 → 错了。

### 方案二：曼彻斯特编码——让数据自带节奏

每个比特中间必须跳变一次：`1 = 低跳到高 ▁▔`，`0 = 高跳到低 ▔▁`。接收方锁住跳变节奏 → 时钟偏差永远不会积累超过半个比特。

代价：每个数据比特需要 2 次信号变化 → 效率只有 **50%**。

### 方案三：块编码——提高效率

不需要每个比特都跳变，只需要"跳变得足够频繁"。

```
4B/5B 编码（百兆以太网）：
  4 个数据比特 → 映射为 5 个编码比特
  从 32 种组合中挑选 16 种"跳变足够多"的组合
  效率：4/5 = 80%

64B/66B 编码（10Gbps+ 以太网）：
  64 个数据比特 → 66 个编码比特
  效率：64/66 ≈ 97%
```

编码方案的本质就是在"时钟同步的可靠性"和"效率"之间做权衡。从 50% 到 80% 到 97%，每一代技术都在拉近这个权衡的极限。

## 问题 5：用香农定理重新理解三种介质

| | 双绞线（Cat6a） | 光纤（单模） | Wi-Fi 5GHz |
|---|---|---|---|
| **物理带宽 B** | 500 MHz | 数 THz | 160 MHz |
| **噪声来源** | 串扰 + 热噪声 | 极低的热噪声 | 墙壁/金属/微波炉/邻居 Wi-Fi |
| **有效 S/N** | 中等 | 极高 | 低到中（环境决定） |
| **香农 C 上限** | ~10 Gbps | 数百 Tbps | ~数 Gbps |
| **距离限制** | 100m（铜电阻衰减） | 40~80km | 几十米（空气衰减+墙壁） |
| **稳定性** | 稳定（封闭介质） | 极稳定 | 不稳定（开放空间） |

**光纤碾压铜线的根本原因：B 大了几个数量级**，不是 S/N 更好，是 B 直接乘出来的结果。

**Wi-Fi 不稳定的根本原因：开放空间没有屏蔽**。信号走无数条路径（直射、穿墙、反射、绕射），每条路径的延迟不同，到达时叠成一片模糊。

## 总结：一根完整的逻辑链

```
1. 有 0 和 1 要发 → 必须用物理信号表示（电压/光/电磁波）

2. 信号在路上衰减 + 被噪声污染 → 接收方看到模糊波形

3. 香农定理 C = B × log₂(1+S/N)：在这条有噪声的线路上，每秒最多无误传多少比特

4. 要接近香农极限，必须先解决时钟同步 → 编码方案

5. 编码方案在"效率 vs 时钟同步"之间优化：
   曼彻斯特（50%）→ 4B5B（80%）→ 64B66B（97%）

6. 不同介质提供不同的 B 和 S/N：
   光纤 > 铜线（B 大几个数量级），有线 > 无线（封闭空间 S/N 远高于开放空间）
```

> **物理层在说："这是我的极限。"上面六层在说："知道了，我们来兜底。"**
>
> TCP 为什么需要重传？→ 因为热噪声导致误码。为什么需要多跳路由？→ 因为信号衰减，单段链路不能太长。为什么跨洲延迟这么高？→ 因为光速有限。
