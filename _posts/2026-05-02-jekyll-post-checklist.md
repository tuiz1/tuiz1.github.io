每次发新文章前，按这个清单检查一遍。

---

## 坑 1：发布时间设为未来时间

**现象**：文章推送到 GitHub，Actions 构建成功，但页面上找不到文章。

**原因**：Jekyll 默认不显示未来时间的文章。比如今天是 5 月 2 日，你把 `date` 设为 `2026-05-02 22:00:00`，当前时间还没到 22 点 → 文章被当作"未来文章"跳过。

**解决**：把时间设为一个已经过去的时刻。

```yaml
# 错误：还没到晚上 10 点
date: 2026-05-02 22:00:00 +0800

# 正确：当天凌晨，肯定已经过去
date: 2026-05-02 00:00:00 +0800
```

> 也可以只写日期 `date: 2026-05-02`，Jekyll 默认会补 `00:00:00`。

---

## 坑 2：Liquid 引擎吃掉花括号

**现象**：文章包含 `{ }` 花括号（比如集合 `{0, 1}`、数学表达式），本地看起来没问题，但部署后内容混乱或页面报错。

**原因**：Jekyll 使用 Liquid 模板引擎，`{...}` 和 `{% ... %}` 会被当成模板指令解析。你的数据里的花括号被当成了 Liquid 代码 → 要么报错，要么内容被吞。

**解决**：在每篇文章的正文前后加上 `{% raw %}` 和 `{% endraw %}`，告诉 Liquid 跳过这篇文章的内容。

```markdown
---
title: 我的文章
date: 2026-05-02 00:00:00 +0800
categories: [某分类]
tags: [某标签]
---
{% raw %}

正文内容写在这里。{0, 1} {0, ..., 2^k-1} 随便写，不会被 Liquid 解析。

{% endraw %}
```

**规律**：只要文章中出现了花括号，就加上 `{% raw %}`。

---

## 检查清单

发文章之前：

- [ ] `date` 时间是不是已经过去的时间？
- [ ] 正文有没有花括号 `{ }`？有就加 `{% raw %}` / `{% endraw %}`
- [ ] 文件名格式：`YYYY-MM-DD-slug.md`
- [ ] `categories` 和 `tags` 写好了
- [ ] 代码块用三个反引号，不用 Liquid 的 highlight 标签

---

## 调试技巧

**本地预览**（避免等 GitHub Actions 跑完才知道出错）：

```bash
# 在项目根目录安装依赖
bundle install

# 启动本地服务器
bundle exec jekyll s

# 浏览器打开 http://localhost:4000 查看效果
```

**查看构建日志**：GitHub 仓库 → Actions → 点最近的 workflow run → 展开 Build site → 看有没有报错。

---

## 小结

两个最常见的坑：

| 坑 | 症状 | 修法 |
|---|---|---|
| 发布时间没到 | 文章不显示 | `date` 写 `00:00:00` 或前一天 |
| 没加 raw 标签 | 内容错乱/构建失败 | 正文前后加 `{% raw %}` ... `{% endraw %}` |

每次按清单检查，不会再掉进去了。
